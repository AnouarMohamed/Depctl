# Deployment Kit Contract

## Purpose

The deployment kit is the central product artifact.

It must be readable by humans and executable by the tool.

Default location:

```text
.deploy/
```

## Required structure

```text
.deploy/
  README.md
  detected.json
  plan.json
  docker-compose.yml
  Dockerfile
  .dockerignore
  .env.example
  traefik/
    dynamic.yml
  nginx/
    default.conf
  ci/
    github-actions.yml
  scripts/
    deploy.sh
    rollback.sh
    status.sh
    backup.sh
  reports/
    scan-report.md
    plan-report.md
    validation-report.md
  backups/
```

Only generate folders that are relevant to the selected preset.

For Traefik, do not generate Nginx unless needed.

For Nginx, do not generate Traefik.

## `detected.json`

Machine-readable scan result.

Must contain:

- tool version;
- project root;
- timestamp;
- runtime detections;
- framework detections;
- env vars;
- services;
- existing Docker files;
- confidence scores;
- warnings.

## `plan.json`

The exact plan that `apply` will execute.

Must contain:

- selected preset;
- domain;
- public service;
- generated files;
- required host actions;
- Docker network names;
- Compose project name;
- image names;
- volume names;
- warnings;
- manual steps;
- hash of important generated files if possible.

## Human reports

### `scan-report.md`

Explains what was detected and why.

### `plan-report.md`

Explains what will be generated.

### `validation-report.md`

Explains whether it is safe to apply.

## Generated file header

Each generated file should include a short header when the file format allows comments.

Example:

```yaml
# Generated by depctl
# Source plan: .deploy/plan.json
# Review before applying.
```

Do not insert long noisy comments.

## File overwrite policy

Default:

- write only inside `.deploy/`;
- never overwrite existing user files outside `.deploy/`.

If `.deploy/` already exists:

- keep previous files in `.deploy/backups/`;
- show diff summary;
- allow `--force` only with explicit confirmation.

## Suggested `.deploy/README.md`

Should include:

- detected stack;
- selected preset;
- generated files;
- required manual secrets;
- commands to validate/apply;
- rollback command;
- troubleshooting notes.

## Example plan schema

```json
{
  "version": "0.1",
  "project": {
    "name": "my-app",
    "root": "/srv/my-app"
  },
  "preset": "compose-traefik",
  "domain": "app.example.com",
  "publicService": "web",
  "runtime": {
    "name": "node",
    "framework": "nextjs",
    "confidence": 0.91
  },
  "services": [
    {
      "name": "web",
      "type": "app",
      "build": ".",
      "internalPort": 3000,
      "public": true
    },
    {
      "name": "postgres",
      "type": "database",
      "image": "postgres:16",
      "public": false,
      "volume": "postgres_data"
    }
  ],
  "env": {
    "required": ["DATABASE_URL", "APP_SECRET"],
    "sensitive": ["APP_SECRET"]
  },
  "actions": [
    {
      "type": "compose_up",
      "file": ".deploy/docker-compose.yml"
    }
  ],
  "warnings": [
    "No health endpoint detected."
  ]
}
```

## The big rule

`apply` must only apply `plan.json`.

It must not create a new plan silently.
