# Depctl v1.1.1

## Summary

This release improves first-run CLI UX and provider authentication.

## Highlights

- Adds an embedded Depctl startup banner for interactive terminal sessions.
- Adds `--no-banner` and `DEPCTL_NO_BANNER=1` for automation.
- Updates the local installer to show the Depctl banner after a successful build.
- Changes Vercel deploys to use existing CLI login or start `vercel login` when needed.
- Changes Fly.io deploys to use existing CLI login or start `fly auth login` when needed.
- Keeps `VERCEL_TOKEN` and `FLY_ACCESS_TOKEN` as CI/non-interactive paths, not the default local UX.
